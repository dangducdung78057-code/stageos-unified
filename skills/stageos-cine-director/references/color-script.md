# 色彩脚本（Color Script）

把节目的情绪弧翻译成光色弧。每个叙事段落一组色：主色（服装/屏）、光色温、tint 参数。

## 目录
1. 叙事弧模型
2. 色系库（经调研验证的舞台色系）
3. 屏色脚本（hall-800 专用）
4. 与 scene-light 的参数对接
5. 禁忌

## 1. 叙事弧模型

| 段落 | 占比 | 情绪 | 色彩策略 |
| --- | --- | --- | --- |
| 开场 | 10-15% | 建立、期待 | 低饱和、低色温偏暖或中性，亮度含蓄 |
| 发展 | 30-40% | 推进、积累 | 主色逐步饱和，色温随情绪偏移 |
| 高潮 | 15-20% | 顶点 | **对比最大**：饱和度或色温跳变，亮度峰值 |
| 回落/结尾 | 15-25% | 释然、升华 | 回到低饱和但更明亮，或暖金收束 |

规则：高潮段的色彩必须与前一段形成**可感知对比**（Δ色温≥1000K 或 主色色相跳变≥60°），否则高潮立不住。

## 3. 屏色脚本（hall-800 专用）

- 屏色 = 段落主色，喂给 `screen_tint.py --color <hex> --intensity 0.6-0.85` 得人物四区反射；
- 段落切换时屏色渐变过渡 2–4s，禁止硬切（除非高潮点刻意为之）；
- 屏色与服装色**错开 30° 以上色相**，否则人物被背景吃掉；
- 高潮段屏可以降亮度衬人物（屏越亮面光越弱的反向用法：压屏衬人）。

## 4. 与 scene-light 的参数对接

分镜表每段的 light 字段：

```json
"light": { "model": "screen|sky|natural|warm", "kelvin": 3200, "screen_color": "#2E5A88", "intensity": 0.75 }
```

- sky：另需 date/time/venue，由 scene-light 实算阴影；
- screen：screen_color+intensity 直接进 screen_tint.py；
- warm/natural：只取 kelvin 转 key_tint。

## 5. 禁忌

- 全片超过 3 个主色相 → 视觉杂乱，K12 节目 2 个主色+1 点缀色封顶；
- 屏色与服装同色 → 人物隐身；
- 高潮段无色彩对比 → 情绪平顶；
- 低明度服装 + 低亮度屏 + 深色地板三者同时出现 → 画面死黑。
