# StageOS 光照模型

两套模型：室外天空光（太阳位置驱动）、室内分光源（自然光/暖光/大屏彩色光）。输出统一喂给 2.5D 渲染管线：L1 层阴影偏移 + 精灵 multiply tint。

## 1. 室外天空光模型

输入：经纬度（默认南昌 28.68°N, 115.86°E）、日期、时刻。
运行 `scripts/sun_position.py` 得：

- `elevation` 太阳高度角 → 决定影长：`shadow_factor = 1/tan(elevation)`
- `shadow_azimuth` 阴影方位 → 决定阴影投向
- `shadow_vector_25d {dx,dy}` → 直接乘角色高度，作为 L1 层 blob shadow 的偏移向量（等距 y 轴已压扁 0.5）
- `kelvin` / `key_tint` → 键光色温与精灵受光面 tint

规则：
- 高度角 < 10°：金色长影，影长 > 3 倍身高，允许影出画；
- 高度角 > 60°（近正午）：影缩到脚下，几乎不可见，转以脚下接触影为主；
- 幕墙/建筑反射场地（mall-roadshow）：阴影侧加 15–25% 天空蓝灰填充光，避免死黑。

## 2. 时段预设（快捷档，免算脚本时用）

| 预设 | 时刻 | 视觉特征 | 影长 | 色温 |
| --- | --- | --- | --- | --- |
| `morning` | 08:00 | 东侧低角度，暖金，影投向西南偏西，长影 | ~1.5–2.5×身高 | 3200–4300K |
| `noon` | 12:00–13:00 | 近顶光，中性白，影缩脚下 | <0.3×身高 | 5500–6500K |
| `evening` | 17:30（夏）/16:30（冬） | 西侧低角度，暖橙，影投向东，长影 | ~2–4×身高 | 3000–4000K |

精确场景（如指定演出日期）必须用 sun_position.py 实算，预设只用于快速预览。

## 4. 大屏彩色光反射模型（hall-800 专用）

把 LED 主屏当作角色背后的主导彩色面光源，运行 `scripts/screen_tint.py --color <屏色> --intensity <0-1>`，得四区 tint：

| 分区 | 物理含义 | 渲染应用 |
| --- | --- | --- |
| `rim` | 屏光直射角色背部/侧缘轮廓 | 角色背屏侧边缘描边/叠色（混合最强，默认系数 0.85×I） |
| `front_fill` | 屏环境光中和面光 | 正面整体轻微罩屏色（0.25×I） |
| `floor_bounce` | 屏色经地板反射打亮角色下部 | 角色下 1/3 渐变压屏色×地板色（0.45×I） |
| `ambient` | 暗部整体提升 | 阴影区不死黑（0.15×I） |

- intensity 估算：屏面积/亮度占舞台总照明的比例，报告厅主屏常取 0.6–0.85；
- 屏内容换色（如蓝色歌曲→红色歌曲）→ 重跑脚本更新四区 tint，对应收费功能"随屏变色"。

## 5. 与渲染管线的对接参数

```
scene_light = {
  "venue": "hall-800",
  "model": "screen",            # sky | natural | warm | screen
  "key_tint": "#FFEDE3",        # 键光 multiply 色
  "shadow": {                   # 仅天空光模型
    "azimuth": 259.2,
    "vector_25d": {"dx": -1.52, "dy": 0.15}
  },
  "screen_zones": {...}         # 仅大屏模型，screen_tint.py 原样输出
}
```

- 阴影画在 L1 层（blob shadow 按 vector_25d 偏移 + 按 shadow_factor 拉长）；
- tint 走渲染管线 TintCache，键值为 (sprite_id, scene_light_hash)，换场地/换屏色即失效重算；
- 白底精灵（sprite-forge 规范）是所有 tint 的前提，禁止对深色底素材直接 multiply。
