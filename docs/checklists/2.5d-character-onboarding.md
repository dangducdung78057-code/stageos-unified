# 2.5D 新角色接入检查清单

## 1. 建立安全骨架

- [ ] 从 `docs/templates/2.5d-character-manifest.development.json` 复制 Manifest。
- [ ] 目录为 `<characterId>/<variantId>`，`spriteId` 为二者以连字符拼接。
- [ ] 保持 `assetStatus=development`、`placeholder=true`、`productionReady=false`。
- [ ] 填写至少一条尚未完成、可执行的 `productionBlocker`。
- [ ] 未交付的底图与 mask 保持 `null`，不创建透明空图或伪造路径。

## 2. 导入候选素材

- [ ] 三方向底图同时交付，均为相同画布的 8-bit RGBA PNG。
- [ ] 每个启用区域均有三方向 mask；禁用区域保持 `null`。
- [ ] Manifest 只使用角色包内安全相对路径。
- [ ] 校准 `worldHeightCm`、`footBaselineY`，并计算 `anchorY=footBaselineY/canvas.height`。
- [ ] 运行 `pnpm validate:characters`，逐项修复错误，不绕过 Validator。

## 3. 接入 Development 预览

- [ ] 仅在真实资源存在后加入运行时 `SPRITE_MANIFESTS`。
- [ ] 运行时状态仍为 Development，不修改默认 Production 角色解析。
- [ ] 不为单个角色修改 PixiJS 通用方向、换色或 optional-region 逻辑。
- [ ] 增加目标角色的 Manifest、路径、optional region 和序列化测试。
- [ ] 运行 `pnpm verify` 并提交独立的 Development 接入 commit。

## 4. 人工 QA

- [ ] 在 1440 × 900 视口明确选中目标 `performerId/spriteId`。
- [ ] 验证正面、左前、右前三方向无漂移、跳尺、白边或脏块。
- [ ] 使用高对比色逐一验证所有启用区域无串色。
- [ ] 确认禁用区域不显示控件、不渲染、不持久化。
- [ ] 保存到云端、刷新并恢复，确认方向与颜色完整往返。
- [ ] QA 后恢复专用账号状态并清理测试会话和场景数据。
