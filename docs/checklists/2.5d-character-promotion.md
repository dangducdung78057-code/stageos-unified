# 2.5D 角色 Production 晋级检查清单

## 1. 晋级前置条件

- [ ] Development 接入 commit 已合并且 GitHub Actions 成功。
- [ ] `pnpm validate:characters` 通过，三方向与所有启用区域资源完整。
- [ ] `worldHeightCm`、画布、脚底基线和锚点已人工确认。
- [ ] PixiJS 三方向、独立换色、透明边缘和深度关系 QA 通过。
- [ ] 云端保存、刷新和恢复 QA 通过。
- [ ] 所有 Development `productionBlockers` 均有完成证据。

## 2. 状态切换

只允许在独立晋级提交中进行以下状态切换：

- [ ] 静态 Manifest：`assetStatus` 改为 `production`。
- [ ] 静态 Manifest：`placeholder` 改为 `false`。
- [ ] 静态 Manifest：`productionReady` 改为 `true`。
- [ ] 静态 Manifest：`productionBlockers` 清空为 `[]`。
- [ ] 运行时 Manifest 同步为 `production / false`。
- [ ] 更新自动化断言，验证静态与运行时状态一致。

## 3. 禁止事项

- [ ] 不替换或修改其他已封板 Production 角色资源。
- [ ] 不同时修改 PixiJS 通用渲染、方向映射或 optional-region 规则。
- [ ] 不因“文件看似齐全”自动晋级。
- [ ] 不保留已过期阻塞项，不以空透明 PNG 代替禁用区域。
- [ ] 不把 Development 接入和 Production 晋级混在同一 commit。

## 4. 发布证据

- [ ] 运行 `pnpm verify`。
- [ ] 检查 `git diff --check` 和提交范围。
- [ ] 推送晋级 commit 并等待 GitHub Actions 完成。
- [ ] 记录 commit hash、Actions Run ID、CI 结果与 QA 结论。
- [ ] 仅在 CI 成功后宣布 Production 晋级完成。
