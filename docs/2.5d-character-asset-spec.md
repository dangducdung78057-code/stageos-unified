# StageOS 2.5D 角色资产规范

本规范定义 StageOS 2.5D 半写实角色的可复用资产包。`primary-boy-basic-white` 与 `primary-girl-basic-white` 是已验收的生产参考实现；新角色必须先以 development 占位包进入仓库，经素材、自动校验和人工 QA 全部通过后才可晋级。

## 1. 目录结构模板

```text
public/assets/stage-2.5d/characters/
└── <character-id>/
    └── <variant-id>/
        ├── manifest.json
        └── preview/
            ├── front.png
            ├── front-left.png
            ├── front-right.png
            └── masks/
                ├── upper-front.png
                ├── lower-front.png
                ├── footwear-front.png
                ├── accent-front.png              # optional
                ├── upper-front-left.png
                ├── lower-front-left.png
                ├── footwear-front-left.png
                ├── accent-front-left.png         # optional
                ├── upper-front-right.png
                ├── lower-front-right.png
                ├── footwear-front-right.png
                └── accent-front-right.png        # optional
```

空白骨架阶段只提交 `manifest.json`，不得提交伪造图片、透明空图或把不存在的路径接入运行时 Manifest。

## 2. Manifest 模板

```json
{
  "schemaVersion": 2,
  "assetVersion": "draft-YYYYMMDD-1",
  "characterId": "<character-id>",
  "spriteId": "<character-id>-<variant-id>",
  "variantId": "<variant-id>",
  "style": "2.5d-semi-realistic",
  "assetStatus": "development",
  "placeholder": true,
  "productionReady": false,
  "regions": {
    "upper": { "enabled": true },
    "lower": { "enabled": true },
    "footwear": { "enabled": true },
    "accent": { "enabled": false }
  },
  "productionBlockers": [
    "三方向底图未交付",
    "已启用区域的方向遮罩未交付",
    "anchor 与 footBaseline 尚未校准",
    "QA 验收未完成"
  ],
  "worldHeightCm": 150,
  "canvas": {
    "width": 1024,
    "height": 1536,
    "unit": "px",
    "origin": "top-left",
    "footBaselineY": null,
    "anchorX": 0.5,
    "anchorY": null,
    "alignment": "center-bottom"
  },
  "views": {
    "front": { "image": null, "masks": { "upper": null, "lower": null, "footwear": null, "accent": null } },
    "front-left": { "image": null, "masks": { "upper": null, "lower": null, "footwear": null, "accent": null } },
    "front-right": { "image": null, "masks": { "upper": null, "lower": null, "footwear": null, "accent": null } }
  }
}
```

`worldHeightCm` 必须由角色设定确认，模板值不可直接晋级。素材交付后可补充 `headTopY`、`maskEncoding` 与 `defaultColors`。

## 3. 三方向资源规则

- 必须同时提供 `front`、`front-left`、`front-right`，不可用镜像图冒充另一方向。
- 三张底图必须使用相同画布尺寸、透明背景、人物尺度与脚底基线。
- 文件名和 Manifest 路径必须严格一致；方向切换不得改变人物世界身高。
- Production 状态禁止方向回退；development 状态允许路径为 `null`，但不得接入默认角色解析。
- 每个已启用区域必须在三个方向各有一张独立 alpha mask。

## 4. Optional region 规则

标准区域为 `upper`、`lower`、`footwear`、`accent`。区域是否参与渲染由 `regions.<region>.enabled` 唯一决定：

- `enabled: true`：三个方向都必须提供遮罩；QA 面板显示该区域；Production 缺任一遮罩即阻断晋级。
- `enabled: false`：三个方向遮罩必须为 `null`；UI、渲染和持久化均忽略该区域。
- `accent` 通常为 optional；不得通过修改 PixiJS 通用换色逻辑为单个角色特判。
- 遮罩使用白色目标区、透明背景、alpha-mask 合成；不得包含其他区域像素。

## 5. 尺度与锚点规则

- `worldHeightCm`：角色设定中的真实世界身高，必须为正数，并用于 2.5D/3D 尺度统一。
- `footBaselineY`：源图像中双脚接地基线的像素 Y 坐标，不是透明画布底边。
- `anchorX`：默认 `0.5`，表示脚底水平中心。
- `anchorY`：必须按 `footBaselineY / canvas.height` 计算，不手填近似值。
- `anchorX`、`anchorY` 必须处于 0 到 1；三方向共享同一画布、基线和锚点。
- `alignment` 固定为 `center-bottom`，`origin` 固定为 `top-left`。

## 6. Validator 规则

自动校验至少覆盖：

1. JSON 可解析，`schemaVersion`、ID、variant、style 与目录一致。
2. development 骨架必须满足 `assetStatus="development"`、`placeholder=true`、`productionReady=false`，且 `productionBlockers` 非空。
3. Production 必须满足 `assetStatus="production"`、`placeholder=false`、`productionReady=true`、`productionBlockers=[]`。
4. Production 的三方向底图均存在，尺寸一致且有透明通道。
5. 每个 enabled region 的三方向遮罩均存在；disabled region 的路径为 `null`。
6. 遮罩尺寸与底图一致，目标区域白色、背景透明，不跨区污染。
7. `anchorY === footBaselineY / canvas.height`（允许浮点误差），锚点范围合法，`worldHeightCm > 0`。
8. 运行时 `spriteManifestSchema` 可解析，`isProductionReady()` 与静态 Manifest 的 `productionReady` 一致。
9. development 骨架不得进入 `SPRITE_MANIFESTS` 或 `resolveSpriteId()`。
10. `check:encoding`、TypeScript、ESLint、Tests 与 Production build 全部通过。

## 7. QA 验收清单

- 在 1440 × 900 桌面视口使用明确的 performerId / spriteId 选中目标角色。
- 正面、左前、右前三方向逐一切换，确认无跳尺、漂移、白边、矩形脏块或断层。
- 对所有 enabled region 分别使用高对比色，确认独立换色且无跨区串色。
- 确认 disabled region 不显示控件、不渲染、不持久化。
- 检查透明底、脚底基线、同排深度关系、遮挡与世界身高比例。
- 保存到云端，刷新并恢复，确认方向和颜色完整往返。
- 记录提交哈希、测试对象、失败项与人工晋级结论。

## 8. Production 晋级清单

仅在以下项目全部完成后晋级：

- 三方向底图和全部 enabled region 遮罩已交付并通过 Validator。
- `worldHeightCm`、`footBaselineY`、`anchorX/Y` 已人工校准。
- 自动测试、完整质量门及目标角色专项 QA 全部通过。
- 静态 Manifest 与运行时 Manifest 同步更新。
- 将 `assetStatus` 改为 `production`、`placeholder` 改为 `false`、`productionReady` 改为 `true`，并清空 `productionBlockers`。
- 单独提交、推送，等待 GitHub Actions 成功后再打发布标签。
- 不以文件“看似齐全”自动晋级，不修改已封板生产角色或通用 PixiJS/optional-region 规则。

## 9. 批量校验与安全边界

运行以下只读命令扫描全部角色包：

```bash
pnpm validate:characters
```

Validator 会检查目录/ID、状态不变量、三方向、optional region、路径安全、画布和锚点，以及 PNG 的签名、IHDR 尺寸、RGBA 色彩类型。它不会写入、复制、重命名、删除或自动晋级任何文件。

新角色必须从 `docs/templates/2.5d-character-manifest.development.json` 开始，并遵循 `docs/checklists/2.5d-character-onboarding.md`。Production 晋级必须使用 `docs/checklists/2.5d-character-promotion.md`，且必须是独立提交。

## 10. 状态与目录不变量

目录固定为：

```text
public/assets/stage-2.5d/characters/<characterId>/<variantId>/manifest.json
```

并满足：

- `characterId`、`variantId` 必须与目录一致。
- `spriteId` 必须等于 `<characterId>-<variantId>`。
- `views` 必须且只能包含 `front`、`front-left`、`front-right`。
- Development 固定为 `development / true / false`，并至少保留一条可执行阻塞项。
- Production 固定为 `production / false / true`，且 `productionBlockers` 必须为空。
- 禁用区域的三方向 mask 必须为 `null`，不得用空透明 PNG 代替禁用语义。
- 所有资源路径必须是角色包内的相对路径，不得包含绝对路径或 `..` 路径穿越。

为保证已封板 Production 资源零改动，Validator 只对现有 schema v2 历史包兼容 `frontLeft/frontRight` 方向别名，以及早期 Production 包中缺省的 `regions` / `productionBlockers`。新模板不得使用这些缺省或别名；后续接入统一使用连字符方向键并显式声明完整字段。
